# 项目定位、内在逻辑与应用价值

> 回答一个关键问题：这些功能不是"工具堆砌"，而是围绕一条主线、服务两类真实决策的有机整体。

## 一句话定位

**一个以"可达性"为主线、覆盖"平时宜居 + 灾时韧性"两类城市决策的三维空间分析与决策支持平台。**

平时看"去哪儿方便、住哪儿值"，灾时看"哪里会淹、往哪儿逃"——两者建立在同一套数据底座
（三维白模 + 路网 + POI + 水系 + 地形）和同一套空间分析方法（缓冲/邻近、网络分析、空间统计、
栅格连通）之上，方法复用、彼此咬合，而非互不相关的小工具。

---

## 内在逻辑：两个内核 + 一个统一入口

### 内核 A · 平时（日常宜居）：可达性驱动的价值—服务—选址链

```
价值评估(到六类设施的加权邻近度)
      │  揭示空间结构
      ▼
热点分析(Moran's I / Gi*)  ── 价值高值簇/低值簇
      │  对照
      ▼
服务区/等时圈(设施 N 分钟可达) ── 公共服务覆盖与盲区
      │  反向优选
      ▼
选址分析(阈值/Top-K，可按"重点设施"侧重) ── 落到具体地块
      │  日常出行
      ▼
路径规划(多交通方式：驾车/骑行/步行/公交，可设途径点)
```

**为什么是一条链**：可达性同时决定了**地段价值**（离设施越近越值钱）、**公共服务公平性**
（等时圈覆盖）和**选址优劣**（反向筛选），热点分析则刻画其空间格局。多交通方式让"可达性"
对不同人群（有车 / 无车 / 老人步行）分层——同一地块对开车的人很方便、对步行的人可能很差。

### 内核 B · 灾时（安全韧性）：情景化淹没—疏散闭环

```
洪水淹没(强降水致河湖暴涨 / 海平面上升；按高程从水体连通漫淹；涨水过程动画)
      │  产生"危险区"这一空间约束
      ▼
撤离路径(把淹没区当作路网障碍 → 自动避开 → 就近选避难场所；多交通方式)
      │  监测/瞭望覆盖
      ▼
视域分析(考虑建筑高度的三维通视：监控点/瞭望塔能看到哪里)
```

**为什么是闭环**："情景 → 风险 → 响应"——淹没分析的输出（危险区多边形）直接喂给路径规划做
避险疏散，形成联动；涨水过程动画呈现风险随水位升高的演进；按交通方式的疏散可达性区分了
"车行撤离"与"老弱步行撤离"的不同压力。

### 统一入口 · AI 助手

AI（DeepSeek function calling）是覆盖上述所有分析的自然语言入口：
- **降低门槛**：规划人员无需学习操作，用中文描述需求即可；
- **编排多分析**：一次真实决策往往要多个分析协同，AI 支持多轮追问串联
  （"先评估这片的地段价值" →"再看 8 米水位会不会淹" →"那从这里规划步行撤离"）——
  这正是"内在逻辑"的体现：分析之间是接力，不是并列。

## 两个内核为何是"同一城市系统的两面"

高可达、设施密集的城区，往往也是灾时高风险、高疏散压力区。平台让**平时价值/可达性**与
**灾时风险/疏散能力**在**同一个三维场景**中对照，回答"我们投入建设的高价值地区，是否也做好了
韧性准备"——这是把宜居与安全统一思考的城市治理视角。

## 实际应用场景

| 场景 | 用到的模块组合 | 决策产出 |
|------|----------------|----------|
| 房地产 / TOD 选址 | 价值评估 + 服务区 + 交通方式 + 选址 | 可达性好、公共服务全的住宅/商业优选地块 |
| 公共服务补点 | 服务区(等时圈) + 热点 | 医院/学校 15 分钟覆盖盲区 → 新设施选址 |
| 城市体检 | 价值评估 ↔ 服务区 对照 | 找"高价值低服务"或"低价值高服务"的错配区 |
| 防汛应急预案 | 淹没(情景+涨水动画) + 撤离 + 视域 | 受淹小区/道路、避难所与疏散路线、监测盲区 |
| 灾时实时响应 | 淹没危险区 → 撤离重算 | 按当前淹没范围动态生成避险路线 |

## 创新点（凝练）

1. **可达性统一主线**：价值、热点、服务区、选址、路径共享"邻近/可达"内核，互相印证而非堆砌。
2. **"平时宜居 + 灾时韧性"双内核同台**：在同一三维平台对照人居价值与灾害风险/疏散能力。
3. **淹没—疏散闭环联动**：淹没区作为路网障碍驱动避险路径重算；**涨水过程动画**动态推演风险。
4. **多交通方式贯穿**可达性与疏散，支持按人群（车行/步行/骑行/公交）分层评估公平性。
5. **AI 自然语言编排多分析、多轮追问**，贴合真实决策流程，显著降低专业工具门槛。
6. **工程上数据与算法解耦**：离线样例开箱即用，一键 `fetch_data.py` 切换真实 OSM，引擎零改动。

## 现状与后续（数据真实性）

- 当前**路网为真实 OSM**（已抓取），**POI / 水系暂用内置样例**（Overpass 在国内被限流，
  抓取返回 0 要素；平台已自动回退样例，保证可演示）。
- 要让价值/选址/热点/淹没落在**真实地物**上：配置代理后重跑 `fetch_data.py` 抓到真实 POI/水系，
  重启后端即自动启用——平台的分析逻辑、创新点完全不变，只是把"样例底图"换成"真实底图"。
- 若要更严谨的淹没，可放入真实 `dem.npy`（高程），淹没即基于真实地形而非合成地形场。
