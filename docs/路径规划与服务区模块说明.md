# 路径规划、灾害撤离与服务区/等时圈模块说明

本文用于说明本人负责的后端模块：通勤路径规划、灾害撤离路径规划，以及服务区/等时圈分析。该模块的核心目标是基于真实路网完成“点到点最优路径”“就近安全撤离”“设施 N 分钟可达范围”三类网络分析，并在需要时调用高德/百度在线服务获得含实时路况的结果。

## 1. 模块定位

本模块承担项目中的“城市路网网络分析”任务，主要包括四个接口：

| 功能 | 接口 | 主要后端文件 |
|---|---|---|
| 通勤路径（内置路网） | `POST /api/route` | `backend/app/engine/routing.py` |
| 通勤路径（高德在线，含实时路况/公交地铁） | `POST /api/route/amap` | `backend/app/engine/amap.py` |
| 灾害撤离路径 | `POST /api/evacuate` | `backend/app/engine/routing.py` |
| 服务区 / 等时圈 | `POST /api/service-area` | `backend/app/engine/service_area.py`、`backend/app/engine/baidu.py` |

它们共享同一套“路网图 + 最短路”的底层能力：

```text
道路网 roads.geojson
  -> 构建为 NetworkX 加权图（边权 = 距离 / 通行时间）
  -> 通勤路径：起讫点间最优路径
  -> 撤离路径：从起点就近选安全、可达、容量足的避难场所
  -> 服务区：从设施点出发，各时间档可达范围成面
```

在“内置路网”之外，本模块还提供两条在线增强通道：高德（通勤，含实时路况与公交/地铁）与百度（服务区，含实时路况校准），离线能力始终作为兜底。

## 2. 通勤路径（内置路网）

### 2.1 功能目标

在研究区路网上规划起点到终点的最优路径，支持多种交通方式、途经点，以及“最快 + 最短”多条备选。

### 2.2 输入参数

接口：

```text
POST /api/route
```

请求参数：

```json
{
  "city": "hangzhou_core",
  "start": [120.16, 30.27],
  "end": [120.20, 30.24],
  "optimize": "time",
  "mode": "drive",
  "vias": [[120.18, 30.26]],
  "alternatives": true,
  "hazard": null
}
```

| 参数 | 含义 |
|---|---|
| `start` / `end` | 起讫点 `[lon, lat]`（WGS84） |
| `optimize` | `time`（最快）或 `length`（最短） |
| `mode` | `drive` / `cycle` / `walk` / `transit`，影响通行时间 |
| `vias` | 可选途经点列表，路径依次经过 |
| `alternatives` | 为 `true` 时同时返回“最快 + 最短”两条备选（去重） |
| `hazard` | 可选 GeoJSON 几何，路径将避开它（用于避险绕行） |

`optimize` 与 `mode` 用 `Literal` 做了取值校验，非法值直接返回 422；起讫点坐标做了定长校验，避免空坐标导致 500。

### 2.3 核心算法

路网在内存中构建为 `NetworkX` 无向加权图：节点为道路折点（经纬度按 6 位小数取整以共享端点），边权同时保存“长度（米）”与“通行时间（分钟）”。最短距离用长度做权，最快用通行时间做权。

通行时间按交通方式估算：步行/骑行/公交用固定平均速度，驾车用道路自身限速。考虑到离线路网没有实时路况，驾车额外乘了一个**城市拥堵系数**（实际平均车速约为自由流限速的一定比例），避免“半小时开出几十公里”的高估。

起讫点通过 KDTree 吸附到最近路网节点。为避免研究区外的点被静默吸附到边界道路、返回看似正常实则错误的路线，吸附设置了**最大距离阈值**（约 3km），超出则提示“点不在研究区或离路网过远”。

多备选时分别用“最快”和“最短”两套权重各算一条，并按距离/时间去重。

### 2.4 输出结果

输出为 GeoJSON `FeatureCollection`，每条路径是一个 LineString，`properties` 含 `length_m`、`time_min`、`optimize`、`mode`、`rank`、`strategy_cn`；`meta` 含 `mode_cn`、`vias`、`alts`（各备选的距离/时间摘要）。

## 3. 通勤路径（高德在线）

### 3.1 功能目标

“内置路网”只能找出最短路，找不出“用时最少”的真实路况路径。为对齐高德/百度地图的体验，本模块提供高德在线代理：驾车返回含实时路况的多条备选，并支持杭州的公交/地铁换乘。

接口：`POST /api/route/amap`（前端勾选“用高德在线路线”或选择“公交/地铁”时调用）。

### 3.2 坐标转换

高德使用 GCJ-02（火星坐标系），本项目内部统一用 WGS84，二者在国内偏移约 500m，必须转换。`amap.py` 实现了 `wgs84_to_gcj02` / `gcj02_to_wgs84`，请求前把起讫点转为 GCJ-02，返回的折线再转回 WGS84，经实测往返误差在米级。

### 3.3 关键实现

- 驾车走高德路径规划 v5，开启 `alternative_route=3` 返回多条备选（与高德网页版一致，例如“最快 / 躲避拥堵 / 距离最短”）。
- 公交/地铁走高德 transit/integrated 接口，按 `city=330100`（杭州 adcode）查询，**分段返回**步行/地铁/公交段（含线路名与配色），可在地图上按交通方式区分上色。
- 公交/地铁目前仅支持杭州研究区；东京等海外区域前端会自动回退到内置路网近似，避免直接调用高德失败。

### 3.4 配置

高德 Key 通过 `backend/.env` 的 `AMAP_KEY` 注入；未配置时该选项报错，但不影响内置路网。

## 4. 灾害撤离路径

### 4.1 功能目标

从起点前往**安全、可达、容量足**的最优避难场所，并可避开危险区（如洪水淹没多边形）。相比只挑“最近”，更贴近应急决策。

接口：`POST /api/evacuate`。

### 4.2 实现逻辑

```text
1) 危险区（hazard）做 ~80m 缓冲，模拟道路封控/积水边缘，删除其上的道路；
2) 排除落在危险区内的避难所（自身被淹的避难所不可用）；
3) 对每个候选避难所计算可达时间，按“可达时间 + 容量”综合评分择优。
```

综合评分对时间和容量分别归一化后加权（时间权重 0.6、容量权重 0.4），并在结果 `reason` 中给出推荐依据（在多少个安全可达避难所中综合最优、多少分钟可达、容量多少）。撤离起点同样做了吸附距离校验。

### 4.3 输出结果

输出包含撤离路线（LineString）与目标避难所点；`meta` 含 `shelter`、`capacity`、`time_min`、`candidates`、`mode_cn`、`reason`。

## 5. 服务区 / 等时圈

### 5.1 功能目标

从设施中心点出发，计算各时间档（如 5/10/15/20/30 分钟）的可达范围，用于评估服务覆盖能力。提供两种数据源：

- **离线（默认）**：本地路网可达节点成面，全城市可用、无外部依赖；
- **百度实时路况（仅国内）**：用百度批量算路校准真实耗时，更贴近现实，海外或失败自动回退离线并注明。

接口：`POST /api/service-area`，请求体含 `center`、`bands`、`mode`、`baidu`（是否启用百度）。

### 5.2 离线成面（凹包 + 水体裁剪）

```text
本地路网单源 Dijkstra（按通行时间，含拥堵系数）得到可达节点
  -> 每个时间档取“可达时间 <= 档位”的路网节点
  -> 凹包（concave hull）成面，贴合可达点云
  -> 用真实水体数据（water.geojson）从面里 difference 掉湖泊/河流
```

为什么用凹包而不是“逐道路段缓冲”：真实路网驾车大范围可达边段可达十万级，逐段缓冲并集会超时；凹包是一次性运算、零超时。为控制计算量，对节点做了采样限点。湖泊/河流（如西湖）通过真实水体多边形裁剪挖空，而不是靠缓冲留洞，避免产生零散不合理的碎洞。

> 局限：当前没有真实 DEM/地类数据，**山体暂不能裁剪**；这属于后续工作（需补一层 DEM 或 `natural` 地类）。

### 5.3 百度实时路况校准

策略是“百度校准耗时，本地路网成面”，慢一点但稳定、图形贴合路网：

```text
1) 本地 Dijkstra 得到可达节点的离线耗时；
2) 按时间分层抽样若干节点，串行调用百度批量算路（Route Matrix v2）取真实耗时；
3) 用“真实/离线”耗时的中位数比值，校准全部节点耗时（全局拥堵系数）；
4) 每个时间档用校准后耗时成面，覆盖请求的全部档位，绝不静默丢档；
5) 百度有效样本不足/被限流时离线补全，并在 meta.note 明示。
```

关键约束：所有百度请求**串行**（绝不并发），批次间限速，遇到 `status=401`/“并发超配额”按 2/4/8s 退避重试——低配额账号主要受 QPS/并发限制，而非日额度。坐标以 WGS84 传入（`coord_type=wgs84`），无需反向转换。百度 AK 通过 `backend/.env` 的 `BAIDU_AK` 注入，并需在百度控制台开通“批量算路”。

### 5.4 输出结果

输出为各时间档的面 Feature，`properties` 含 `minutes`、`area_km2`、`reachable_nodes`，百度模式额外含 `source`、`sampled_points`、`baidu_valid_points`；`meta` 含 `bands`（请求档位）、`summary`、`mode_cn`、`provider`（`offline` / `baidu` / `offline_fallback`）、`calib_ratio`、`note`。前端按 `meta.bands` 展示全部请求档位，缺档显示“未生成”，不会被返回结果掩盖。

## 6. 与前端的交互方式

前端通过 `frontend/src/api/index.js` 调用后端，开发环境由 Vite 代理到 `http://localhost:8000`。交互链路：

```text
ControlPanel.vue
  -> App.vue 的 runRoute / runEvacuate / runServiceArea
  -> frontend/src/api/index.js
  -> FastAPI 接口（/route、/route/amap、/evacuate、/service-area）
  -> 后端网络分析引擎（routing / amap / baidu / service_area）
  -> 返回 GeoJSON
  -> cesium/layers.js 渲染路线、撤离、等时圈（含孔洞/边界线）
  -> ResultPanel.vue 展示距离/时间/备选/服务区档位与来源
```

前端主要依赖字段：路线的 `length_m`、`time_min`、`alts`、`legs`（公交分段）；撤离的 `shelter`、`capacity`、`time_min`、`reason`；服务区的 `bands`、`summary`、`provider`、`note`。后端迭代需保持这些字段稳定，避免破坏渲染。

## 7. 本次改进

本次改进集中在“让 ③④ 两类网络分析既能离线稳定运行，又能对齐高德/百度的真实路况体验”。

1. **通勤对齐在线地图**：新增高德在线代理（含 WGS84↔GCJ-02 转换），驾车支持多条实时路况备选；新增杭州公交/地铁分段换乘。
2. **驾车更贴近现实**：内置路网驾车引入城市拥堵系数，修正“速度远高于实际”的高估。
3. **撤离更合理**：从“就近”升级为“可达时间 + 容量”综合评分，并排除被淹避难所、避开危险区。
4. **服务区两种数据源**：离线（凹包 + 真实水体裁剪）与百度实时路况校准（串行 + 限速 + 退避，绝不并发、绝不丢档、样本不足明示）。
5. **健壮性**：起讫点/中心点加最大吸附距离校验（研究区外的点不再被静默吸附）；接口 `mode`/`optimize` 用 `Literal` 校验、坐标定长校验（空坐标不再 500）；分析失败时清空旧图层与旧结果，避免误导。
6. **可读性**：组合交通方式分段配色；服务区等时圈各档分明配色、保留并描绘水体孔洞；结果面板显示来源（provider）与说明（note）。

## 8. 给前端/队友的接入建议

- 通勤面板：勾选“用高德在线路线”走 `/route/amap`，否则走 `/route`；“公交/地铁”非杭州研究区自动回退内置近似。
- 服务区面板：勾选“用百度实时路况”走百度（仅国内、需配置 `BAIDU_AK`），否则离线；面板按 `meta.bands` 渲染全部档位。
- 在线服务请放在后端代理，AK 写入 `backend/.env`（已 `.gitignore`），不要提交密钥，也不要把 AK 放到前端。
- 与本模块无关的文件（如 `flood.py`、`value.py`、`stats.py`、`fetch_data.py`）不在本 PR 范围内，避免与其他同学的 PR 冲突。

## 9. 可放入报告的总结表述

本模块基于真实路网完成城市路网网络分析，覆盖通勤路径、灾害撤离与服务区/等时圈三类功能。路网在内存中构建为加权图，以距离或含拥堵系数的通行时间为边权，支持多交通方式、途经点与“最快/最短”多备选；起讫点吸附设最大距离阈值以保证结果可信。为对齐真实路况体验，通勤可经高德在线代理返回含实时路况的多备选与杭州公交/地铁分段换乘（含 WGS84↔GCJ-02 坐标转换）。灾害撤离以“可达时间 + 容量”综合评分择优，并避开危险区与被淹避难所。服务区/等时圈提供离线（凹包成面 + 真实水体裁剪）与百度实时路况校准（串行限速调用批量算路、以真实/离线耗时比值校准、本地路网成面、样本不足离线补全）两种数据源，结果以 GeoJSON 向前端输出，供 Cesium 三维渲染与结果面板展示。
