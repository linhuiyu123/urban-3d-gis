# 城市三维可视化与分析平台 · 技术说明文档

> 面向课程大作业的完整技术说明。涵盖系统架构、数据流、各空间分析方法的原理与实现、
> 接口规范、运行指南，以及与三门课程知识点的对应关系。

## 1. 系统架构

平台采用经典的 **前端 — 后端 — 数据** 三层 + 大模型增强的架构：

```
┌─────────────────────────────────────────────┐
│ ① 前端表现层  Vue3 + Vite + CesiumJS          │
│   三维白模 · 分析面板 · AI对话框 · 玻璃拟态UI    │
└───────────────┬─────────────────────────────┘
                │  HTTP / REST · GeoJSON
┌───────────────▼─────────────────────────────┐
│ ② API 服务层  FastAPI                         │
│   /analysis /route /stats /service-area       │
│   /flood /ai 等接口，pydantic 校验             │
└───────────────┬─────────────────────────────┘
                │
┌───────────────▼─────────────────────────────┐
│ ③ 空间分析引擎                                 │
│   GeoPandas/Shapely · NetworkX · PySAL · NumPy │
│   + DeepSeek（function calling）              │
└───────────────┬─────────────────────────────┘
                │
┌───────────────▼─────────────────────────────┐
│ ④ 数据层  杭州为主 / 东京备用                   │
│   Cesium OSM Buildings(白模) · POI · 路网       │
└─────────────────────────────────────────────┘
```

**为什么这样分层？** 前端只负责"可视化与交互"，所有计算密集的空间分析放后端，
既符合 WebGIS 的 B/S 设计范式（应用开发课 01、04、11 讲），也便于把同一套分析引擎
同时暴露给"人工操作面板"和"AI 助手"两个入口。

## 2. 数据流（以"价值评估"为例）

1. 用户在前端点击"运行价值评估" → `api.value(city)` 发出 `POST /api/analysis/value`；
2. 后端 `routers/api.py` 用 pydantic 校验请求 → 调 `engine/value.assess_value()`；
3. 引擎从 `data_loader` 取 POI（文件缺失时由 `sample_data` 程序化生成）；
4. 投影到 UTM → 计算每个网格到各类设施的最近距离 → 距离衰减 + 加权叠加 → 归一化；
5. 返回 GeoJSON 网格（每格带 score 与分项得分）；
6. 前端 `layers.LayerManager.renderValueGrid()` 把网格渲染为按分数着色、按分数抬升的三维价值面。

AI 入口的差别仅在第 1 步：自然语言 → DeepSeek 选择工具与参数 → 同样调第 3 步之后的引擎。

## 3. 各分析方法原理与实现

### 3.1 住房 / 地段价值评估（缓冲 + 距离衰减 + 加权叠加）

对六类设施（景点 scenic、商业 commercial、学校 school、医院 hospital、公交 transit、主干道 road），
每个网格点计算"邻近度得分"：

```
邻近度 s_cat = exp( - d / decay_cat )
```

- `d`：网格点到该类最近设施的距离（米，UTM 平面，scipy.cKDTree 加速最近邻查询）；
- `decay_cat`：该类的特征影响距离（半衰减尺度，相当于"软缓冲区"半径），
  如商业 800m、公交 500m、学校 600m。距离越近得分越接近 1，越远越接近 0。

综合价值 = 各类邻近度的加权和，再线性归一化到 0–100：

```
V = Σ ( w_cat · s_cat ) / Σ|w_cat|     →  归一化到 [0,100]
```

权重 `w_cat` 可调（默认商业 0.25、交通/学校 0.20…）。**选址分析**就是对该结果阈值筛选：
保留 `V ≥ 阈值` 的网格，或取分数最高的前 K 个。

> 对应：空间分析"邻近分析/缓冲区分析""叠置分析"，导论 Lect.10、应用开发 13 讲（矢量空间分析）。

### 3.2 路径规划（网络分析）

把道路 GeoJSON 构建为 NetworkX 加权图：节点=道路折点，边权有两种——
路段长度（米）与通行时间（分钟 = 长度 /（限速·1000/60））。

- **通勤路径**：`nx.shortest_path` 按 `time` 或 `length` 求最优路径；
- **撤离路径**：遍历所有避难场所，取从起点"时间最短可达"者，再求最短路；
- **避险绕行**：传入危险多边形（如洪水淹没范围）时，先删除与之相交的边，再在子图上求路。

> 对应：导论 Lect.15"网络分析"、应用开发 13 讲。

### 3.3 空间统计：热点 / 空间自相关

- **全局 Moran's I**：度量价值在空间上是否聚集。I>0 表示高值/低值各自成团（正自相关），
  I≈0 近似随机，I<0 表示高低相邻（负自相关）。
- **局部 Getis-Ord Gi\***：对每个网格给出 z 分数，`z>1.96` 为热点（高值簇），
  `z<-1.96` 为冷点（低值簇），其余不显著（95% 置信）。

实现优先用 PySAL（`esda` + `libpysal`，课程推荐生态）；若环境未装，自动降级到本项目内置的
等价 NumPy 实现（KNN 权重 + 公式直算），保证功能始终可用。

> 对应：空间分析 Lect.5（空间相关）、Lect.8（聚类分析）。

### 3.4 服务区 / 等时圈

从设施点出发，用 `nx.single_source_dijkstra` 按"时间"求单源可达性，
按 5/10/15 分钟阈值取出可达节点集合，凸包 + 缓冲平滑成等时圈多边形，前端嵌套着色。

> 对应：网络分析中的"服务区分析"。

### 3.5 洪水淹没模拟（水文连通 + 平滑水面）

不是简单"凡低于水位的格子都淹"，而是模拟真实涨水：

1. **从水体起淹**：以真实水系（河 / 湖 / 海岸线，OSM）为种子，用广度优先搜索（BFS）
   只向"与水体连通且低于水位"的相邻格子扩散——内陆孤立低洼若不与水体连通，不会被淹。
   - 有真实 DEM（`dem.npy`）时，用真实高程做 `高程 < 水位` 判据；
   - 无 DEM 但有真实水体时，用"离水体距离"作高程代理（每 100m 距离抬升 1m，约 1% 坡度），
     淹没从河岸 / 海岸向外成**带状**扩散，水位越高带越宽。
     注意：**种子只取落在水体内的格子**，扩散范围由水位控制——否则固定宽度的种子带会盖过
     水位的作用，导致不同水位淹没范围几乎一样（已修复）。
2. **平滑水面**：淹没格融合后做形态学平滑（buffer 外扩再内收）+ 拓扑简化，
   输出**单一连续的水面多边形**，告别"小方块拼贴"的粗糙观感。

融合后的淹没范围回传作为撤离避让多边形，形成"淹没 → 自动避险重算撤离"的闭环。

> 对应：导论 Lect.9（栅格数据处理）、应用开发 14 讲（栅格空间分析）。

### 3.6 三维专属分析：视域 / 天际线 / 日照

- **三维视域 / 通视**：从观察点按"方位角 × 俯仰角"发射射线（默认 96 方位 × 4 俯仰 ≈ 384 条），
  用 `scene.pickFromRay` 对三维白模做求交。由于射线带俯仰分量、白模有真实高度，
  能体现高楼对视线的**三维遮挡**；观察高度（楼层）可调，绿色=可视、红色=被挡；
- **天际线**：按建筑高度对白模分级着色（高→暖、低→冷），呈现城市天际轮廓；
- **日照阴影**：开启 `viewer.shadows`，时刻滑块步长 0.1h 连续驱动太阳，阴影丝滑变化；
  可选"夜晚加深"——按时刻降低底图亮度与大气亮度，夜景更真实。

> 对应：导论 Lect.14（VR/AR 与新技术）、应用开发 06–08 讲（Cesium 三维可视化）。

### 3.7 AI 空间分析助手（大模型 + 空间分析）

将上述七项分析注册为 DeepSeek 可调用的"工具"（OpenAI function calling 规范）：

1. 用户自然语言提问（可携带"当前点选坐标"作为上下文）；
2. DeepSeek 判断调用哪个工具、填哪些参数（如城市、阈值、坐标）；
3. 后端本地执行该分析，得到 GeoJSON；
4. 把结果摘要回传模型，生成一句中文解释；
5. 前端据返回的 `kind` 把图层高亮联动到地图上。

未配置 `DEEPSEEK_API_KEY` 时，自动降级为基于关键词的 Mock 规则引擎，保证联动链路可演示。

> 对应：导论 Lect.11"人工智能与空间分析实践"（大模型 + 空间分析、智学GIS 的思路）。

## 4. 接口规范（REST）

| 方法 | 路径 | 说明 | 主要参数 |
|------|------|------|----------|
| GET  | `/api/cities` | 城市元数据 + POI 类别中文 | — |
| GET  | `/api/data/pois\|roads\|shelters` | 基础数据 | `city` |
| POST | `/api/analysis/value` | 价值评估网格 | `city, weights?, resolution?` |
| POST | `/api/analysis/site` | 选址筛选 | `city, min_score, top_k?` |
| POST | `/api/route` | 通勤路径 | `city, start, end, optimize, hazard?` |
| POST | `/api/evacuate` | 撤离路径 | `city, start, hazard?` |
| POST | `/api/stats/hotspot` | 热点/自相关 | `city` |
| POST | `/api/service-area` | 等时圈 | `city, center, bands` |
| POST | `/api/flood` | 洪水淹没 | `city, water_level` |
| POST | `/api/ai` | AI 助手 | `prompt, city, context_point?` |

所有返回均为 GeoJSON（或其包装），坐标系 WGS84（EPSG:4326）。启动后访问
`http://localhost:8000/docs` 可交互测试全部接口。

## 5. 运行指南（Windows 11 / PowerShell）

**后端**

```powershell
cd backend
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
.\.venv\Scripts\python.exe -m uvicorn app.main:app --reload --port 8000
```

> 一行一行回车。直接用虚拟环境里的 python 启动，免激活，也能避开 PATH 上其它环境的
> `uvicorn.exe` 报 `Fatal error in launcher: Unable to create process`。
> 想激活：`.\.venv\Scripts\Activate.ps1`（被拦先 `Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass`，
> 或 CMD 里用 `.venv\Scripts\activate.bat`）。
> 运行时**只需** fastapi/uvicorn/numpy/scipy/shapely/networkx/pyproj；`geopandas`/`fiona`/`osmnx`、
> `esda`/`libpysal`、`openai` 均为可选，装不上不影响启动（PySAL 降级 NumPy、AI 降级 Mock）。

**前端**

```powershell
cd frontend
npm install
npm run dev      # http://localhost:5173
```

**配置**（已预置你的 Key）：`backend/.env` 内 DeepSeek Key；`frontend/.env` 内 Cesium ion 令牌。

**数据**：默认用内置样例数据（离线即跑）。如需真实数据，在 PowerShell 中分两步执行
（PowerShell 5.1 不支持 `&&`，请勿写在同一行）：

```powershell
cd data
python fetch_data.py hangzhou_core
```

> 安装提示：`geopandas`、`esda/libpysal` 在 Windows 下若 pip 安装困难，建议用
> conda：`conda install -c conda-forge geopandas esda libpysal osmnx`。
> 即便 PySAL 缺失，后端也会自动降级，不影响演示。

## 7. 设计取舍与可拓展点

- **样例数据 vs 真实数据**：默认程序化生成样例数据保证开箱即用；`fetch_data.py` 可一键换成
  OSM 真实数据，引擎代码无需改动（数据与算法解耦）。
- **可拓展**：已预留接口，便于追加"一键分析报告导出（Word/PDF）""GWR/回归房价预测"
  "时空动态（不同时段通勤热力）"等功能。
- **性能**：价值网格默认 48×48、淹没 100×100，兼顾精度与前端渲染流畅度；可经 `resolution` 调整。

## 8. v2 修订说明（针对反馈的改进）

| 反馈问题 | 改进措施 |
|----------|----------|
| 范围太小（只有西湖方块） | 研究区改为**可切换多档**：杭州主城 / 都市区 / 全域 + 东京（`config.AREAS`） |
| 路网差、只有一条线、偏移 | 路径规划改用**真实 OSM 路网**（`fetch_data.py` 经 OSMnx 抓取主干+次干道） |
| 选点没有标识 | 点选后在地图放置**醒目立柱+标签标记**（`LayerManager.marker`） |
| 卫星图与底图偏移 | 底图统一使用 **WGS84 源**（影像/带注记影像/OSM街道），不混用 GCJ-02，天然对齐 |
| 底图单一、无地名 | 顶栏新增**底图切换**：影像注记 / 纯影像 / 街道（含地名） |
| 洪水死板、方块拼贴 | 重做为**从真实水系起淹 + 连通扩散 + 单一平滑水面**（见 3.5） |
| 视域仅二维 | 升级为**考虑建筑高度的三维视域**（方位×俯仰射线，观察高度可调，见 3.6） |
| 日照不丝滑、夜晚不黑 | 滑块步长 0.1h **连续**；新增**夜晚加深**可选项 |
| AI 交互少、无上下文 | 新增**执行步骤展示 + 运行动画**；后端维护**多轮会话上下文**（`SESSIONS`） |
| UI 简陋、参数不可调 | 每模块加**说明与原理提示**；价值**权重系数**、视域**高度/半径**等参数均可调 |

> 数据获取（PowerShell，分两行）：`cd data` 后再 `python fetch_data.py hangzhou_core`
>（按研究区分别抓取；全域数据量大，建议优先抓主城/都市区）。未抓取时后端自动使用内置样例，离线可演示。

## 9. v3 修订说明（本轮 Bug 修复与体验优化）

| 反馈问题 | 根因 | 改进措施 |
|----------|------|----------|
| 路径规划偏移严重 | 路线用固定 30m 绝对高度渲染，开地形 + 倾斜视角产生视差 | 路线/点改 `clampToGround` / `CLAMP_TO_GROUND` 贴地；最近节点吸附按 cos(纬度) 校正 |
| 淹没分析不合常理 | 固定宽度的种子带盖过水位，水位 1~14m 淹没几乎不变 | 种子只取水体内格子，扩散范围由水位控制；分辨率 70→100、坡度代理 1% |
| 价值评估整片“全蓝” | 衰减尺度（数百米）相对数十公里研究区过小，邻近度≈0 | **衰减尺度随研究区对角线自适应**；归一化改 2/98 分位裁剪 |
| 选址 0 个达标 | 同上（价值整体被压低） | 随价值修复一并解决；新增「重点设施」按类型侧重选址 |
| 淹没位置不真实 | 内置水系是 bbox 对角直线河（占位） | 合成水系改为**近中心湖泊 + 带弯河流**；可用 `fetch_data.py` 换真实水系 |
| AI 无法使用 | 走 DeepSeek 分支时若 `openai` 未装或调用失败会 500 | **任何失败优雅降级为本地规则引擎**并提示；确保 `openai` 已安装即用在线模型 |
| 热点分析运行失败 | PySAL 装上但运行期版本不兼容、且无回退 | PySAL 调用包 try/except，**运行期出错也回退内置 NumPy 实现** |
| 可达区域区分不明显 | 等时圈各档同色、低透明度叠加 | 各时间档**绿→红分明配色 + 描边**，结果面板列出各档可达节点数 |
| 街道底图叠地形很奇怪 | 街道瓦片贴在三维起伏上被拉伸 | **地形显示改为可选开关**，切到街道底图自动关闭地形 |
| 参数/结果展示不足 | — | 价值/淹没新增**分辨率可调**；结果面板补充均值/分位/面积/可达节点等；按钮按前置条件禁用 |

> 真实数据：本轮已让内置合成数据离线即合理；如需地理真实结果，运行 `data/fetch_data.py`
> 抓取真实 OSM 路网/水系/POI 覆盖样例（生成 `data/<区域>/*.geojson`，**重启后端**生效）。
